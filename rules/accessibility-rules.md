# Accessibility Rules

Every block produced by this framework, and every page generated from those
blocks, must meet WCAG 2.2 AA. Generation agents are responsible for
producing accessible output the first time; the evaluator catches regressions.

---

## 1. Semantic structure

- Each page has exactly one `<h1>`. Subsequent headings descend in order
  (`h2`, then `h3`, never `h1` followed by `h3`).
- Section blocks render a `<section>` element with an `aria-labelledby`
  attribute pointing to the block's heading. The `Hero`, `FeatureSystem`,
  `Pricing`, and `Conversion` blocks in `packages/blocks` enforce this.
- Navigation uses `<nav aria-label="Primary">`. A second nav (e.g. footer)
  uses a distinct label.
- Lists of features, plans, or testimonials use `<ul>` / `<li>`. Visual
  card grids do not exempt the markup from being a list.

## 2. Color and contrast

- Body text against background: contrast ratio ≥ 4.5:1.
- Large text (≥ 18.66 px bold or 24 px regular): ≥ 3:1.
- Non-text contrast for icons, focus rings, and form borders: ≥ 3:1.
- Never communicate state through color alone. Status pills must include an
  icon or text label in addition to color.

## 3. Focus

- Every interactive element has a visible focus state. shadcn/ui's
  `focus-visible:ring-ring` is the default; do not override it to remove the
  ring.
- Focus order matches visual order. When using `flex-row-reverse` or
  `order-*` utilities, set explicit `tabIndex` only as a last resort and
  document why.
- Modal and dialog blocks trap focus and restore it on close.
- Skip-to-content link is rendered as the first focusable element on every
  page generated by the framework.

## 4. Images and icons

- Every `<img>` has an `alt` attribute. Decorative images use `alt=""`.
  No `alt="image"` or `alt="logo"`.
- Icons that convey meaning include an `aria-label` or visible text;
  decorative icons set `aria-hidden="true"`.

## 5. Motion and timing

- Respect `prefers-reduced-motion: reduce`. When using **Framer Motion**, call
  `useReducedMotion()` and skip animations (static layout) or snap to final
  values; CSS-only blocks may use `motion-reduce:` / `motion-safe:` as a
  secondary guard.
- No content moves automatically for more than 5 seconds. Carousels — if
  used at all — are user-controlled.
- No flashing content above 3 Hz.

## 6. Forms

- Every form control has an associated `<label>`. Placeholder text is not a
  substitute for a label.
- Error messages are programmatically associated via `aria-describedby` and
  conveyed in text, not only color.
- Required fields use `aria-required="true"` in addition to any visual
  marker.

## 7. Touch targets

- Interactive targets are at least 44 × 44 CSS pixels on touch viewports.
- Adjacent interactive targets are separated by at least 8 px.

## 8. Language and direction

- The root `<html>` element sets `lang`. The studio app sets `lang="en"` by
  default; generated pages should inherit and override only when the page
  is in another language.
- Layout uses logical properties (`ms-`, `me-`, `ps-`, `pe-`) where
  available so that an RTL audit is a future-friendly change.

## 9. Evaluator coverage

`packages/evaluation/responsiveness-check.ts` runs a static a11y pass
covering:

- Heading order
- Required `alt` attributes
- Form-label association
- Color contrast (computed against shadcn/ui CSS variable tokens)
- Single `<h1>` per page
- Skip-link presence

A failure on any item fails the build.
